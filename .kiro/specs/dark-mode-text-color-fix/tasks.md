# 实施计划: 深色模式文字颜色修复

## 概述

本实施计划旨在系统性地修复Streamlit多Agent竞品分析系统在深色模式下的文字颜色显示问题。修复将分别处理三个关键文件(`app_beautiful.py`、`app.py`、`styles.css`)中的CSS样式,确保所有文本元素在深色模式下显示为白色或浅色,并保持与背景的良好对比度。

修复策略采用渐进式方法:首先修复主要文件`app_beautiful.py`,然后是`app.py`,最后是`styles.css`,每个文件修复后进行验证,确保深色模式和浅色模式都能正常工作。

## 任务

- [ ] 1. 修复 app_beautiful.py 的深色模式文字颜色
  - [ ] 1.1 定位并检查现有深色模式媒体查询
    - 在`app_beautiful.py`中找到`@media (prefers-color-scheme: dark)`媒体查询块
    - 检查现有规则是否覆盖了所有文本元素类型
    - 识别缺失的选择器和不完整的颜色规则
    - _需求: 5.1, 8.1_
  
  - [ ] 1.2 添加通用文本元素的深色模式规则
    - 在深色模式媒体查询内添加通用选择器规则: `.main *`, `p`, `span`, `div`, `li`, `label`, `strong`, `small`
    - 为所有通用文本选择器设置 `color: #FFFFFF !important`
    - 确保使用 `!important` 声明符以保证优先级
    - _需求: 1.1, 1.3, 2.1, 2.2, 4.1_
  
  - [ ] 1.3 添加标题元素的深色模式规则
    - 为所有标题元素(`h1`, `h2`, `h3`, `h4`, `h5`, `h6`)添加 `color: #FFFFFF !important`
    - 为标题的所有子元素添加规则: `h1 *`, `h2 *`, `h3 *`, `h4 *`, `h5 *`, `h6 *`
    - 保持标题的其他样式属性(如 font-weight)不变
    - _需求: 1.2, 2.2, 4.2_
  
  - [ ] 1.4 添加 Streamlit 组件的深色模式规则
    - 为 `.stMarkdown`, `.stMarkdown *`, `.stMarkdown p`, `.stMarkdown li`, `.stMarkdown span` 添加白色文字规则
    - 为 `.stText`, `.stText *` 添加白色文字规则
    - 为指标组件添加规则: `[data-testid='stMetricLabel']`, `[data-testid='stMetricValue']`, `[data-testid='stMetricDelta']`
    - 为按钮组件添加规则: `.stButton > button`
    - 为选项卡组件添加规则: `.stTabs [data-baseweb='tab']`, `.stTabs [aria-selected='true']`
    - 为 Alert 组件添加规则: `.stAlert`, `.stInfo`, `.stSuccess`, `.stWarning`, `.stError` 及其子元素
    - 所有规则使用 `color: #FFFFFF !important`
    - _需求: 1.4, 1.6, 1.7, 2.2, 4.3, 10.1, 10.2, 10.3_
  
  - [ ] 1.5 添加数据表格的深色模式规则
    - 为 `.dataframe`, `.dataframe *`, `.dataframe th`, `.dataframe td` 添加白色文字规则
    - 可选:为表格背景添加深色背景色以提高对比度
    - _需求: 1.5, 4.5_
  
  - [ ] 1.6 添加输入元素的深色模式规则
    - 为 `input`, `textarea`, `select`, `label` 添加 `color: #FFFFFF !important`
    - 为输入元素添加深色背景: `background-color: rgba(50, 50, 70, 0.95) !important`
    - 确保输入框在深色模式下可读且美观
    - _需求: 4.4, 9.4_
  
  - [ ] 1.7 添加自定义类的深色模式规则
    - 为自定义类添加白色文字规则: `.main-title`, `.main-title *`, `.subtitle`, `.subtitle *`, `.info-card`, `.info-card *`, `.feature-badge`, `.success-box`, `.success-box *`
    - 所有规则使用 `color: #FFFFFF !important`
    - _需求: 4.8_
  
  - [ ] 1.8 添加或调整深色模式背景颜色
    - 确保 `.main` 容器有深色背景渐变,例如: `background: linear-gradient(135deg, #1a1a2e 0%, #16213e 50%, #0f3460 100%)`
    - 为 `.element-container` 添加深色背景: `background: rgba(40, 40, 55, 0.95)`
    - 为进度容器和页脚添加深色背景规则
    - _需求: 3.3, 9.1, 9.2, 9.3, 10.4, 10.5_
  
  - [ ] 1.9 验证 app_beautiful.py 的深色模式修复
    - 在浏览器中启用深色模式,打开应用并导航到使用 `app_beautiful.py` 的页面
    - 手动检查所有文本元素是否为白色或浅色
    - 检查文本与背景的对比度是否清晰可读
    - 切换回浅色模式,确认浅色模式样式未受影响
    - _需求: 1.1-1.7, 3.1, 3.2, 6.1, 6.3_

- [ ] 2. 修复 app.py 的深色模式文字颜色
  - [ ] 2.1 定位并检查 app.py 中的深色模式媒体查询
    - 在`app.py`中找到`@media (prefers-color-scheme: dark)`媒体查询块
    - 如果不存在,创建新的深色模式媒体查询块
    - 检查现有规则覆盖范围
    - _需求: 5.1, 8.1_
  
  - [ ] 2.2 复用 app_beautiful.py 的选择器规则到 app.py
    - 参考 `app_beautiful.py` 中已修复的选择器列表
    - 在 `app.py` 的深色模式媒体查询内添加相同的选择器规则
    - 确保文本颜色值一致: `color: #FFFFFF !important`
    - 确保背景颜色方案与 `app_beautiful.py` 保持一致
    - _需求: 1.1-1.7, 2.1, 2.2, 4.1-4.8, 7.1, 7.2, 7.3, 7.4_
  
  - [ ] 2.3 根据 app.py 特有组件添加额外选择器
    - 识别 `app.py` 中使用但 `app_beautiful.py` 未使用的组件和类
    - 为这些特有组件添加深色模式规则
    - 保持与整体风格一致
    - _需求: 4.9, 7.4_
  
  - [ ] 2.4 验证 app.py 的深色模式修复
    - 在浏览器中启用深色模式,打开应用并导航到使用 `app.py` 的页面
    - 手动检查所有文本元素是否为白色或浅色
    - 检查与 `app_beautiful.py` 的视觉一致性
    - 切换回浅色模式,确认浅色模式样式未受影响
    - _需求: 1.1-1.7, 3.1, 3.2, 6.1, 6.3, 7.1_

- [ ] 3. 修复 styles.css 的深色模式文字颜色
  - [ ] 3.1 定位并检查 styles.css 中的深色模式媒体查询
    - 在`styles.css`中找到`@media (prefers-color-scheme: dark)`媒体查询块
    - 如果不存在,创建新的深色模式媒体查询块
    - 检查现有规则覆盖范围
    - _需求: 5.1, 8.1_
  
  - [ ] 3.2 添加全局深色模式文本颜色规则到 styles.css
    - 参考前两个文件中的选择器规则
    - 在 `styles.css` 的深色模式媒体查询内添加全局文本颜色规则
    - 确保所有选择器使用 `color: #FFFFFF !important`
    - 确保背景颜色与前两个文件一致
    - _需求: 1.1-1.7, 2.1, 2.2, 4.1-4.8, 7.1, 7.2, 7.3, 7.4_
  
  - [ ] 3.3 处理 styles.css 中的特殊样式类
    - 识别 `styles.css` 中定义的特殊样式类和ID选择器
    - 为这些选择器添加深色模式规则
    - 确保不与 `app_beautiful.py` 和 `app.py` 的规则冲突
    - _需求: 4.9, 7.4_
  
  - [ ] 3.4 验证 styles.css 的深色模式修复
    - 在浏览器中启用深色模式,全面测试应用的所有页面和组件
    - 确认所有文本元素在所有页面上都为白色或浅色
    - 检查三个文件的样式是否协调一致
    - 切换回浅色模式,进行最终的浅色模式验证
    - _需求: 1.1-1.7, 3.1, 3.2, 6.1, 6.3, 7.1, 7.5_

- [ ] 4. 最终验证和质量检查
  - [ ] 4.1 执行全面的深色模式测试
    - 在现代浏览器(Chrome, Firefox, Safari, Edge)中测试深色模式
    - 遍历应用的所有页面、组件和功能
    - 验证所有文本元素(标题、段落、按钮、标签、表格、Alert等)都为白色
    - 确认没有遗漏的黑色文字
    - _需求: 1.1-1.7, 4.1-4.9_
  
  - [ ] 4.2 执行浅色模式回归测试
    - 在浅色模式下测试应用的所有页面
    - 确认浅色模式下文本颜色未受影响(仍为黑色或深色)
    - 验证浅色模式的背景和其他样式保持不变
    - _需求: 6.1, 6.2, 6.3, 6.4_
  
  - [ ] 4.3 验证深色模式响应性
    - 在应用运行时,切换操作系统或浏览器的深色模式设置
    - 确认样式立即更新,无需刷新页面
    - 测试深色模式与浅色模式之间的来回切换
    - _需求: 5.2, 5.3, 5.4_
  
  - [ ] 4.4 验证对比度合规性
    - 使用浏览器开发者工具或对比度检查工具(如WebAIM Contrast Checker)
    - 测量白色文字(#FFFFFF)与深色背景的对比度
    - 确认所有文本与背景的对比度 >= 4.5:1 (WCAG AA标准)
    - 如有对比度不足的情况,调整背景颜色
    - _需求: 3.1, 3.2, 3.3, 3.4_
  
  - [ ] 4.5 验证 CSS 规则优先级
    - 使用浏览器开发者工具检查计算样式(Computed Styles)
    - 确认所有深色模式文本颜色规则都使用了 `!important` 声明
    - 验证深色模式规则覆盖了其他可能的样式规则
    - 检查是否有优先级冲突导致规则未生效
    - _需求: 2.1, 2.2, 2.3, 2.4, 8.3_
  
  - [ ] 4.6 验证多文件一致性
    - 对比 `app_beautiful.py`, `app.py`, `styles.css` 三个文件的深色模式规则
    - 确认文本颜色值一致(都为 #FFFFFF)
    - 确认背景颜色方案协调一致
    - 确认所有文件都使用 `!important` 优先级
    - 确认选择器命名和结构保持一致
    - _需求: 7.1, 7.2, 7.3, 7.4, 7.5_
  
  - [ ] 4.7 处理动态生成的 Streamlit 组件
    - 测试动态加载的组件(如条件渲染的组件)
    - 确认深色模式规则也应用到动态生成的元素
    - 如有问题,添加更广泛的通配符选择器(如 `.main *`)
    - _需求: 8.4_

- [ ] 5. 检查点 - 确保所有测试通过
  - 确保所有手动测试通过
  - 确认深色模式和浅色模式都能正常工作
  - 如有任何问题,请向用户询问

## 注意事项

- 本修复为纯 CSS 样式修复,不涉及 JavaScript 或 Python 代码修改
- 所有修复都应在 `@media (prefers-color-scheme: dark)` 媒体查询内进行
- 务必使用 `!important` 声明符确保深色模式规则的优先级
- 修复过程中要保持浅色模式的样式不变
- 每个文件修复后都应进行独立验证,避免遗漏问题
- 对比度检查是确保可访问性的关键步骤,不可跳过
- 三个文件的深色模式实现应保持一致,便于后续维护
